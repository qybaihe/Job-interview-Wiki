# 科大讯飞｜语音 AI 平台｜后端开发｜Question Bank

3. 语音流式上传时，服务端为什么通常要区分首帧、持续帧、尾帧？
4. 流式 ASR 场景中，如果网络抖动导致音频帧延迟、重复甚至乱序到达，服务端如何做容错？
5. 为什么语音识别结果通常要区分中间结果和最终结果？
6. 如果要支持识别结果的增量纠正，客户端协议与后端状态管理该怎么配合？

## 二、长连接治理与平台稳定性

7. 语音识别服务为什么不能只按“请求数”限流，还要看并发连接数、音频时长和单连接持续时间？
8. 为什么 WebSocket 长连接治理往往比普通短 HTTP API 更难？
9. 用户反馈“识别结果总是晚半拍”，你会如何拆解客户端录音、网络传输、服务端解码、模型排队这几段延迟？
10. 语音平台为什么需要单独监控连接时长分布、帧间隔、尾帧缺失率、转写完成率？
11. 某次版本上线后识别服务 CPU 飙高但 GPU 没明显变化，可能是什么原因？
12. 如果某租户突然高并发上传长音频，如何在不影响其他租户的前提下进行隔离和降级？
