# 用友｜平台研发｜Java｜question-bank

> 本轮未稳定获取可逐题回链的“用友平台研发 Java 真实面经”公开单帖，因此采用 **公开产品/平台能力域 + 企业软件平台研发高相关问题** 的保守构造方式。
> 题目主要锚定用友官网公开可见的 `用友BIP`、`iuap平台`、`YonBuilder开发平台`、`YonCTP云技术平台`、`YonData数据平台`、`YonAI智能平台`、`YonGPT` 等能力域；不冒充真实单场逐字面经。

## 适用方向
- 平台研发 / Java
- 企业软件平台、中台、PaaS、低代码/开发平台
- 校招 / 实习 / 初中级后端研发

## 一、平台定位与技术选型

1. 你如何理解用友 BIP / iuap 这类企业数智化平台的“平台研发”岗位？它和单一业务线后端开发最大的差别是什么？
2. 如果让你为企业软件平台设计技术底座，哪些能力应该沉淀到平台层，哪些应该留给业务产品层？
3. 为什么企业软件平台往往强调“可配置、可扩展、可集成”，而不只是把接口写出来？
4. 如果一个平台既要服务内部产品团队，也要服务客户二开和生态伙伴，你会如何设计平台边界？
5. 用 Java 做平台研发时，什么样的模块适合抽成公共能力，什么样的模块不应该过早平台化？

## 二、低代码 / 开发平台 / 元数据驱动

6. 用友官网公开展示了 `YonBuilder开发平台`。如果你做低代码/开发平台，元数据驱动通常会驱动哪些对象：表单、流程、权限、校验、页面还是接口？
7. 元数据驱动平台最大的技术收益是什么？最大的风险又是什么？
8. 如果一个低代码平台支持动态表单和动态模型，你会如何设计底层元数据存储与版本管理？
9. 为什么很多低代码平台后期会遇到“灵活但难治理”的问题？你会怎样给平台加约束？
10. 如果客户通过平台自定义字段、流程和页面，如何保证升级后不破坏旧配置？
11. 低代码平台里，运行时解释执行和编译生成代码各有什么优缺点？
12. 如果你要设计平台脚本扩展能力，怎样防止脚本把核心系统拖慢或破坏安全边界？

## 三、云原生平台与微服务底座

13. 用友官网公开出现 `YonCTP云技术平台`。如果平台走云原生路线，你理解的核心底座能力应该有哪些？
14. 平台研发为什么通常会被追问服务治理、配置管理、注册发现、网关和限流熔断？
15. 如果平台上承载很多业务服务，你会如何做统一日志、监控、链路追踪和告警？
16. 在企业软件平台里，为什么“稳定上线”和“灰度发布”通常比“新技术试验”更重要？
17. 如果某个公共平台组件升级会影响几十个业务服务，你会如何设计发布与回滚策略？
18. 对一个 Java 微服务平台来说，线程池、连接池、缓存和消息堆积为什么都需要纳入统一治理？

## 四、多租户 / 权限 / 安全 / 审计

19. 企业软件平台为什么天然要考虑多租户？
20. 多租户隔离常见有库级、Schema 级、表级三种方案，你会如何做权衡？
21. 如果同一套平台服务中既有大客户独享诉求，也有中小客户共享诉求，你会如何设计租户架构？
22. 平台研发里为什么权限模型常常比普通 CRUD 系统复杂得多？
23. 如果平台支持组织、角色、岗位、数据权限、字段权限，你会如何避免权限模型失控？
24. 为什么企业软件平台普遍需要审计日志？什么操作必须留痕？
25. 如果客户管理员能在平台中做配置和二开，怎样保证其操作既可追溯又不影响系统安全？
26. 你会如何设计平台级单点登录、令牌鉴权和会话管理？

## 五、数据平台 / 集成 / 一致性

27. 用友官网公开展示 `YonData数据平台`。如果做数据平台，采集、开发、治理、应用四层通常分别解决什么问题？
28. 企业软件平台为什么往往需要打通 ERP、财务、人力、供应链、CRM、OA 等多系统？
29. 如果你来设计统一集成平台，连接器、重试、幂等、死信和补偿分别应该怎么考虑？
30. 企业软件系统的接口为什么常强调“幂等”和“可追账”？
31. 如果订单、库存、财务凭证跨多个服务流转，为什么最终一致性设计会比单库事务更常见？
32. 你会如何设计一个面向企业内部/生态伙伴的开放 API 平台？
33. 如果客户历史系统数据质量很差，平台接入时你会如何做校验、映射和错误隔离？
34. 当数据同步延迟导致业务和报表结果不一致时，你会如何排查与治理？

## 六、AI 平台与企业软件结合

35. 用友官网首页公开展示 `YonAI智能平台` 与 `YonGPT`。如果在企业软件里接入 AI，平台层最先要补的不是模型能力，而可能是什么？
36. 为什么企业场景里的 AI 平台必须特别重视权限、知识边界、审计与结果可追溯？
37. 如果一个企业智能助手要接入财务、人力、供应链等系统，你会如何做工具调用与权限校验？
38. AI 平台输出错误结论时，为什么企业软件比普通内容产品承受的风险更高？
39. 如果让你在现有 Java 平台里接入大模型能力，你会如何设计模型网关、限流、缓存和降级？

## 七、平台工程实践与项目表达

40. 你做平台研发项目时，最值得在面试中强调的是哪些能力：抽象建模、公共能力沉淀、性能优化、稳定性治理还是跨团队协作？
