# CRON_QUALITY_PLAN.md

## 当前目标

把 `jobwiki-v2` 从“能跑”切换到：

> **围绕 50 家公司扩张 Todo 做高并发扩公司。**

当前阶段最重要的不是继续深挖少数老公司，而是：
- **先把公司数量做大**
- **先覆盖互联网大厂 / 中大厂**
- **每个线程一次只打一家公司**
- **一次性开更多线程并发推进**
- **每轮尽量有新增公司正式提交**

---

## 新运行策略

### 调度节奏
- 继续按 15 分钟一轮
- `schedule.everyMs = 900000`

### 单轮执行预算
- orchestrator 总预算：`720 秒`
- 单线程建议预算：`300 ~ 480 秒`
- 预留收口与 git 提交时间：`120 ~ 180 秒`

### 核心原则
1. **优先扩公司，不优先补老公司**
2. **每个线程一次只做 1 家公司**
3. **多线程并发开工**
4. **允许用高质量 question-bank 作为新公司首个正式入口**
5. **每轮尽量至少拿到 1 个新增公司正式结果**
6. **没有拿到新增公司时，必须如实汇报，不准把老公司补强包装成扩公司成功**

---

## 线程策略：从 5 线程升级为 10 线程

当前建议采用：
- 1 个 orchestrator
- 10 个并行 worker（worker-A ~ worker-J）

### 原则
- 每条线程本轮只能主攻 **1 家公司**
- 优先从 `TODO_COMPANY_EXPANSION.md` 的未覆盖公司中挑
- 多线程同时推进，快速铺开公司池

## 每个 worker 的强制产物

在 `.tmp-runs/<timestamp>-<worker>/` 下至少提供：

1. `report.md`
2. `sources.txt`
3. 候选内容文件：
   - `question-bank.md` 片段，或
   - `interview-experiences/*.json` 候选
4. `target-path.txt`
5. `checklist-update.md`

如果缺少这些核心产物，不得报 `success`。

---

## 新公司“完整有效结果”标准

### 对当前阶段，question-bank 是最优先破冰手段
新公司 question-bank 候选被视为“完整有效”需满足：
- 至少新增 `>= 15` 条非重复问题
- 问题必须能映射到公开来源
- 题目要有明确主题聚类
- 能直接作为新公司的首个正式入口落库

### interview JSON 候选被视为完整有效需满足
- 岗位上下文完整
- 轮次 / 题目 / 结果 / 来源说明完整
- 结构符合 schema
- 不是空壳 JSON

### 当前阶段优先级
对于未覆盖公司：
1. 高质量 `question-bank.md`
2. 真实 JSON
3. 题库 + JSON 同时具备

不要为了追 JSON 错过整轮新增公司机会。

## Orchestrator 的新职责

orchestrator 现在最重要的职责不是“挑最好看的候选”，而是：

1. **先检查主仓库是否已有未提交的新公司正式内容**
2. **若有，优先提交收口**
3. **若无，同时启动 10 条扩公司线程**
4. **确保每条线程只打一家公司**
5. **优先采纳新增公司结果，而不是老公司补强**
6. **本轮尽量收口 1~3 家新增公司正式结果**
7. **提交前确认不是只有 README / progress_state 轻改**

---

## 本轮成功判定

满足任一即可：
1. 新增 1 家此前未覆盖公司，并落至少 1 个正式方向文件
2. 新增 2 家及以上公司候选包，且至少 1 家完成正式落库
3. 多线程形成候选包，最终成功提交至少 1 家新公司正式内容

---

## 本轮失败判定

以下任一情况必须视为失败或部分完成：
- 所有线程都只是在补老公司
- 没有新增公司正式结果
- 只有 README / progress_state / 轻治理修改
- 大多数线程同时处理多家公司但都没收口
- git push 失败

---

## 严格禁止的事

- 禁止把老公司补强包装成“新增公司成功”
- 禁止 1 条线程同轮处理多家公司
- 禁止为了追单个强来源在同一轮无限重试
- 禁止只改统计文件就当作本轮成功
- 禁止把 `.tmp-runs` 候选包当正式产出

## 结论

这套 cron 当前要做的，不是“让少数公司更漂亮”，而是：

> **一次性开很多线程，每个线程打一家公司，快速把 50 家公司基数铺起来。**

在 40~50 家公司基数没建立起来之前：
- 新公司覆盖 = 主线
- 老公司深挖 = 次线
- 治理类任务 = 兜底
