## 一、操作系统与进程线程

1. 进程、线程、协程的差异是什么？系统软件里通常为什么更常直接面对线程模型？
2. 用户态和内核态切换为什么有成本？在系统工具软件里哪些场景会放大这种成本？
3. 什么是上下文切换？如果线程数很多但 CPU 利用率不高，为什么仍然可能出现明显卡顿？
4. Windows 或 Linux 下多线程程序死锁怎么排查？如果现场无法稳定复现，你会优先保留哪些信息？
5. 物理地址和虚拟地址的差异是什么？系统软件工程师为什么需要理解这层抽象？
6. 进程调度算法常见有哪些？为什么有些后台常驻程序即使“逻辑不复杂”也会被调度问题放大体验缺陷？

## 二、C / C++ 基础与资源管理

7. C 语言里 const、volatile、static 分别有什么作用？如果在 BIOS / 驱动 / 底层模块里用错，风险是什么？
8. 常量指针和指针常量分别是什么？各自更常见于哪些接口设计场景？
9. C++ 智能指针各自适合什么场景？shared_ptr 滥用会带来哪些问题？
10. RAII 的核心价值是什么？为什么系统软件尤其依赖它来管理句柄、锁、文件和设备资源？
11. 如果一个模块频繁崩溃，你会如何区分是越界访问、悬空指针还是并发读写问题？
12. crash dump 常见分析思路是什么？最先看哪些信息最有价值？

## 三、设备、驱动与系统级协同

13. 驱动与用户态程序通信通常有哪些方式？各自适合什么场景？
14. 设备管理软件如何识别不同硬件型号，并适配不同能力开关？
15. 如果设备插拔频繁，客户端如何保证状态同步和 UI 展示一致？
16. 蓝牙、USB、网络设备等多类外设并存时，系统软件层最容易出现哪些资源竞争问题？
17. 写一个设备驱动时，设备树（或等价硬件描述）通常解决什么问题？
18. 为什么很多设备管理工具要维护一套本地缓存状态，而不能每次都实时探测硬件？

## 四、安装、升级、驱动/BIOS 更新链路

19. 安装器 / 升级器如何保证幂等？为什么“重复安装不会把系统搞坏”很重要？
20. 自动更新模块通常包含哪些核心能力：版本检测、灰度、断点续传、校验、回滚？
21. 如果升级过程中程序被强制终止，如何保证下次启动仍能自恢复？
22. 驱动、BIOS、应用程序三类更新在风险控制上有什么本质差异？
23. 为什么系统软件的升级链路往往比普通 App 更严格？
24. 如果设备固件升级失败，客户端应如何做回滚、恢复提示和故障隔离？

## 五、稳定性、性能与可观测性

25. 后台常驻程序如何降低 CPU、内存和磁盘占用？
26. 什么是句柄泄漏？和内存泄漏相比，Windows 客户端里它有什么不同表现？
27. 如果线上出现高占用、卡死、黑屏或蓝屏投诉，你会如何做分层排查和止损？
28. 日志采集如何避免影响性能与用户隐私？
29. 为什么“能跑”不等于“能长期稳定跑”？系统软件稳定性通常看哪些指标？
30. 如果让你设计一个设备管理客户端，你会如何拆分 UI 层、业务层、设备接入层和升级维护层？

## 六、算法 / 手写与岗位追问

31. 手撕双向链表时，你会如何实现插入、删除、查找，并保证边界条件正确？
32. 数组去重有哪些做法？如果面试官限制不能直接依赖哈希表，你会怎么权衡时间和空间复杂度？
33. 快排和冒泡排序的原理、时间复杂度、适用场景分别是什么？
34. 如果一个客户端工具要在资源受限环境下长期运行，你会优先优化启动时间、常驻内存还是峰值性能？为什么？
35. 你怎么理解联想 PC 终端生态里的系统软件价值？为什么这类岗位既要求底层能力，也要求对真实设备和用户场景有理解？
