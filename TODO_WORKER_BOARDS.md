# TODO_WORKER_BOARDS.md

## 当前阶段总目标

这份文档专门给 `jobwiki-v2` 的 orchestrator 使用。

当前阶段不是继续围绕少数老公司做深挖，而是：

> **围绕“50 家公司扩张 Todo”做大规模并发扩公司。**

执行原则已经切换为：
- **每个线程一次只做 1 家公司**
- **多线程并发**
- **优先新公司首个正式落库**
- **先把公司数量基数做大**
- **已有公司补深度仅作为兜底，不再是主线**

---

## 全局执行规则（强制）

1. **每个 worker 一次只负责 1 家公司**，禁止同轮贪多
2. **优先未覆盖公司**，除非本轮新公司全部受阻，才允许回头补老公司
3. **允许先用高质量 question-bank 破冰**，不强求首轮必须拿真实 JSON
4. **所有候选先落 `.tmp-runs/<ts>-<worker>/`**，worker 不得直接 commit / push
5. **每个 worker 必须交完整候选包**：
   - `report.md`
   - `sources.txt`
   - 候选 question-bank 或 JSON
   - `target-path.txt`
   - `checklist-update.md`
6. **orchestrator 每轮优先采纳 1~N 个“新增公司”结果**，而不是采纳老公司补强
7. **若本轮没有新增公司成功，也必须明确写清楚哪些新公司 blocked / no-source / partial**

# Orchestrator 调度要求（强制）

## 每轮必须优先做的事
1. 先看主仓库是否已有未提交的“新增公司正式内容”
2. 若有，优先收口提交
3. 若无，则同时启动多条扩公司线程
4. 每条线程只分配 1 家公司
5. 优先从 50 家 Todo 中的未覆盖公司池里挑

## 每轮不允许做的事
- 不允许 1 条线程同时做多家公司
- 不允许大部分线程回头补老公司
- 不允许因为一个线程卡住，就把整轮都耗在该线程上
- 不允许把老公司补强包装成本轮“扩公司成功”
