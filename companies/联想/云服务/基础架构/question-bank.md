# 联想｜云服务 / 基础架构｜Question Bank

> 说明：本题库作为联想在仓库中的首个正式入口，基于工作区内已沉淀的联想候选题库与可确认的岗位方向线索整理，当前定位为 **question-bank**，不冒充单场真实面经逐字还原。
>
> 当前更适合的岗位画像：云服务、基础架构、平台研发、后端基础设施。

## 一、操作系统与网络基础

1. 进程、线程、协程分别解决什么问题？在高并发基础设施服务里你会如何选择？
2. select、poll、epoll 的区别是什么？为什么高并发网络服务通常更偏向 epoll？
3. Linux 一次网络数据包从网卡到应用层大致经历哪些阶段？零拷贝优化了什么？
4. 用户态和内核态切换为什么有成本？在哪些基础架构场景里这个成本会被明显放大？
5. TCP 三次握手和四次挥手各自解决什么问题？为什么会出现 TIME_WAIT？
6. 如果线上出现大量 TIME_WAIT 或 CLOSE_WAIT，你会分别从哪些方向排查？

## 二、后端基础与并发控制

7. 线程池核心参数有哪些？如果基础服务线程池被打满，你会如何定位是 CPU 型、IO 型还是下游阻塞？
8. synchronized、ReentrantLock、CAS 各自适合什么场景？为什么 CAS 不能解决所有并发问题？
9. 什么是幂等？在“控制面下发任务、Agent/节点执行任务”的架构里，哪些操作最需要幂等保护？
10. 如果一个任务调度平台支持失败重试，你如何避免重复执行带来的副作用？
11. 说一下限流、熔断、降级、隔离分别是什么，它们在平台型服务里应该如何组合使用？
12. 如果一个基础服务 P99 延迟突然升高，但 CPU 利用率不高，你会怎么查？

## 三、数据库、缓存与一致性

13. MySQL B+ 树索引为什么适合范围查询？联合索引最左匹配原则在真实 SQL 优化里怎么体现？
14. 事务四大特性是什么？可重复读一定能完全解决幻读吗？为什么很多人容易把它说绝对？
15. Redis 适合解决哪些问题？什么场景下不应该优先想到 Redis？
16. 缓存击穿、穿透、雪崩分别是什么？如果发生在元数据服务上，为什么会更危险？
17. 你如何设计缓存与数据库一致性方案？延时双删、旁路缓存、订阅 binlog 各有什么优缺点？
18. 如果一个配置中心既要求高可用，又要求较强一致性，你会如何在 MySQL、Redis、etcd 之间做选型？

## 四、分布式系统与平台能力

19. 分布式 ID 方案有哪些？数据库自增、Redis、Snowflake 分别有什么问题？时钟回拨怎么处理？
20. 服务注册与发现一般包含哪些核心机制？心跳、租约、健康检查、摘除分别解决什么问题？
21. CAP、BASE 如何在云平台场景下理解？为什么配置中心、服务发现、审计系统的一致性要求不同？
22. 如果让你设计一个任务调度平台，如何支持延迟任务、优先级、失败重试、分片调度和任务去重？
23. 分布式事务为什么难？2PC、TCC、Saga 各自更适合哪类业务？
24. 多租户系统为什么是云平台绕不开的话题？租户隔离通常体现在哪些层面？
25. 如果某个大租户把共享资源池打满，你会如何设计配额、限流、优先级和资源抢占？

## 五、云服务与基础架构设计

28. 配置下发系统和服务发现系统看起来很像，它们在时效性、一致性、容灾上的诉求有什么区别？

30. 对象存储、块存储、文件存储分别适合哪些业务？为什么它们不能简单互相替代？

## 六、容器与云原生
