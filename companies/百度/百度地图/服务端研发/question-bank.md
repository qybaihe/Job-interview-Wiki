# 题库｜百度 / 百度地图 / 服务端研发

> 来源说明：当前目录下已存在 1 份可核验真实面经 JSON；本题库优先补齐地图 / LBS 服务端岗位的专项问题，便于与通用后端题库区分。后续若补到更多公开原帖，可继续沿该结构扩展。

## 基础能力
1. TCP 三次握手、四次挥手分别是什么？TIME_WAIT 的意义是什么？
2. TCP 和 UDP 的区别是什么？地图服务里的定位上报、导航请求、轨迹回传分别更适合哪类协议？
3. MySQL 索引为什么常用 B+ 树？范围查询和排序时有什么优势？
4. Redis 常见数据结构和使用场景有哪些？缓存穿透、击穿、雪崩怎么处理？
5. Linux 下排查接口 RT 抖动，你通常先看哪些指标和命令？

## 地图 / LBS 业务
6. 什么是 POI？一个 POI 检索服务通常包含哪些核心模块？
7. “附近搜索”如何实现？为什么空间粗筛之后还需要排序？
8. GeoHash、四叉树、R 树分别适合什么场景？
9. 经纬度距离如何计算？球面距离与平面距离近似各适用于什么范围？
10. 如果地图检索要兼顾距离、热度、评分、相关性，排序应该怎么设计？
11. 路径规划里 Dijkstra 和 A* 各适合什么场景？
12. 实时路况更新频繁时，如何平衡写入吞吐与查询延迟？
13. 地图瓦片服务为什么适合缓存？缓存 key 怎么设计？
14. 如果 POI 数据每天有大量更新，如何处理索引刷新和缓存一致性？

## 系统设计 / 高并发
15. 设计一个“查附近餐厅”的后端服务，存储、索引、排序、缓存、分页怎么做？
16. 地图搜索接口 QPS 很高时，如何做本地缓存 + Redis + CDN 多级缓存？
17. 如何设计一个位置上报服务，支撑海量移动端持续写入？
18. 路线规划服务如何避免热点城市或热点路线打爆系统？
19. 如果一个地图接口 TP99 很高但平均延迟正常，你先怀疑什么？
20. 如何做限流、降级、熔断，保证核心查路功能优先可用？

## 项目 / 故障追问
21. 讲一个你做过的高并发服务，瓶颈在哪里，怎么优化？
22. 如果地图结果被投诉“不准”，你会如何判断是数据问题、策略问题还是排序问题？
23. 如果线上接口 RT 飙升，你会优先排查网络、缓存、数据库还是下游依赖？为什么？
24. 如果路线规划结果绕路，你会如何做线上验证与回归分析？
