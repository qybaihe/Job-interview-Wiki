# 题库｜字节跳动 / 抖音电商 / 商业化技术 / 后端开发

## 岗位方向
- 典型后端研发场景：广告/商业化链路、投放平台、素材/计划/预算管理、检索与推荐协同、风控与稳定性保障
- 考察重点通常包括：Java 基础、MySQL/Redis、消息队列、分布式事务、高并发设计、接口稳定性、数据一致性、项目抽象能力

## 高频题
1. 线程池核心参数有哪些？`corePoolSize`、`maximumPoolSize`、阻塞队列和拒绝策略分别影响什么？
2. `synchronized` 和 `ReentrantLock` 的区别是什么？你在业务里会怎么选？
3. `HashMap` 为什么线程不安全？并发场景下为什么常换成 `ConcurrentHashMap`？
4. MySQL 的聚簇索引、回表、覆盖索引分别是什么？
5. Redis 为什么快？除了内存还有哪些原因？
6. 缓存穿透、缓存击穿、缓存雪崩分别是什么？如何在商业化流量峰值中防住？
7. Kafka 为什么适合削峰？消息重复消费怎么处理？
8. 幂等怎么做？接口幂等、消息幂等、任务幂等有什么区别？
9. 线上慢 SQL 你会怎么排查？
10. JVM 内存结构、GC 常见算法、Young GC / Full GC 触发条件分别是什么？

## 项目题
1. 讲一个你做过的高并发接口优化项目，瓶颈在哪里，优化前后指标如何变化？
2. 如果广告投放链路某个下游依赖延迟抖动，你如何保护主链路？
3. 你做过的数据一致性方案里，最终一致性是怎么保证的？
4. 讲一次线上故障：发现、止血、定位、复盘、长期治理各做了什么？
5. 如果一个预算扣减服务要求高可用、高一致，你会如何设计？
6. 素材审核/投放计划系统里，如何处理状态机复杂流转？
