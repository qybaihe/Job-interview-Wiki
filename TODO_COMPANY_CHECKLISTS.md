# TODO_COMPANY_CHECKLISTS.md

## 目标

给 orchestrator 一份**按公司拆开的完整 TODO 清单**。

用途：
- 每家公司都有独立 checklist
- orchestrator 可以按公司维度协调 worker
- worker 不只是“搜一下”，而是对每家公司都有明确最小交付目标

统一规则：
- 每家公司至少优先完成 1 个高价值方向
- 最低交付优先为：`question-bank.md` 或真实 `interview-experiences/*.json`
- 若公开来源不够，允许先交付候选（candidate + sources + report）

# 给 orchestrator 的协调规则

1. 先优先选择“未覆盖公司”
2. 每轮最多同时推进 5 个主板块 worker
3. 每个 worker 本轮只主攻 1~2 家公司
4. 如果新公司方向公开来源弱，则先建 question-bank，不强凑 JSON
5. 收口时优先采纳：
   - 新公司正式 question-bank
   - 新公司正式 JSON
   - 新公司目录骨架 + 高质量候选
