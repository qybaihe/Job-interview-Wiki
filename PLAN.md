# PLAN.md

## 项目目标

把中文求职场景中的面试经验整理成一个可持续扩展的结构化知识库：

- 第一层按 **公司名称** 组织
- 第二层进入公司后按 **事业群 / 业务线** 组织
- 第三层再细分到 **部门 / 岗位方向**
- 题库与面经并列维护：
  - `question-bank.md`：Markdown 题库
  - `interview-experiences/*.json`：结构化面试经验

---

## 最终结构目标

```text
companies/
  公司名称/
    事业群/
      部门/
        question-bank.md
        interview-experiences/
          xxx.json
```

要求：
- 根目录不要直接铺很多零散文件
- 第一层只出现公司名称
- 用户点击公司后，看到其下各个事业群
- 再往下进入部门和题库/面经

---

## 推进阶段

### 第一阶段：骨架初始化
目标：
- 建立公司 / 事业群 / 部门树状目录
- 建立题库模板与 JSON 面经模板
- 明确来源规则与清洗规则

当前状态：已完成。

### 第二阶段：小样本验证
目标：
- 先选择少量头部大厂、少量岗位方向
- 人工整理一批高质量面经样本
- 验证目录结构、字段设计、题库抽象是否合理

建议优先：
- 字节跳动
- 腾讯
- 阿里巴巴
- 美团
- 拼多多

### 第三阶段：批量扩充
目标：
- 按公司 / 事业群 / 岗位批量扩展
- 逐步沉淀题库
- 将重复题目抽到 question-bank.md
- 将具体经历保留在 JSON 中

### 第四阶段：统一清洗与去重
目标：
- 合并重复问题
- 规范岗位名称
- 规范事业群命名
- 清理低质量、碎片化、无来源的记录

---

## 查找计划

### 1. 先抓真实面经原文
优先寻找：
- 牛客网面经
- 博客园 / CSDN / 掘金 / 知乎专栏等公开面经
- 个人博客 / GitHub 仓库中的面试总结
- 公开论坛中的结构化面试回忆

### 2. 再抽题库
从面经里抽取：
- 高频基础题
- 算法题
- 项目题
- 系统设计题
- 反问题

### 3. 再补业务归类
按内容把面经归类到：
- 公司
- 事业群 / 业务线
- 部门 / 岗位方向

### 4. 最后做感悟沉淀
每份 JSON 除了题目本身，还要保留：
- 每个题目的感悟
- 最终整场面试的感悟

---

## 执行原则

1. 先少量高质量，再批量扩张
2. 先结构正确，再内容堆量
3. 题库和面经必须分离
4. JSON 不写成长文章，重点保留结构化信息
5. Markdown 题库负责可读性，JSON 负责可检索性
