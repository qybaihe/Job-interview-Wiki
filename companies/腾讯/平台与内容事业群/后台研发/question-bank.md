# 腾讯 / 平台与内容事业群 / 后台研发 / 后端开发题库

> 适用范围：腾讯平台与内容事业群、后台研发、后端开发方向的高频面试题整理。
> 
> 当前定位：基于大厂后台研发通用高频题、腾讯系常见工程化要求以及内容平台业务特点做的结构化题库，用于后续继续补真实面经与反向生成结构化 JSON。

---

## 1. 业务理解与项目题

### 1.1 内容平台项目拆解
- 讲一个你做过的高并发后台项目，核心业务链路、峰值流量和性能瓶颈分别是什么？
- 如果是内容分发、互动、消息、评论、推荐召回等场景，你在系统里承担的具体模块是什么？
- 项目里最难的一次线上事故或稳定性问题是什么？你怎么止损、复盘、避免再次发生？
- 如果业务要求新增一个实时互动功能，你会怎么评估接口、存储、消息链路和容量风险？

### 1.2 指标与业务抽象
- 内容平台常见的核心指标有哪些？播放、停留时长、互动率、转化率分别会影响哪些后端设计？
- 如果业务增长很快，但后端资源成本也同步暴涨，你会从哪些角度做优化？
- 后台研发为什么不能只盯接口 QPS，还要看成功率、延迟分位、消息堆积和回放能力？

## 2. Java / Go / 基础语言能力

### 2.1 Java 基础
- `HashMap` 为什么线程不安全？在并发场景下会出什么问题？
- `ConcurrentHashMap` 在不同 JDK 版本的实现差异是什么？
- 说一下 Java 内存模型、`volatile`、可见性和有序性。
- 什么情况下会触发 Full GC？如何排查频繁 GC？
- 线上 OOM 你一般怎么定位？会看哪些指标和 dump 信息？

### 2.2 Go 基础
- goroutine 和线程的区别是什么？为什么 Go 适合高并发服务？
- channel 适合解决什么问题？它的误用会带来哪些风险？
- Go 中内存逃逸、GC 和对象分配如何影响接口性能？
- 线上 Go 服务延迟抖动明显时，你会怎么排查调度、锁竞争和网络问题？

## 3. 数据库与存储

### 3.1 MySQL
- MySQL 索引失效的典型情况有哪些？
- 联合索引最左匹配原则怎么理解？真实项目里怎么设计复合索引？
- 深分页为什么慢？如何优化？
- 事务隔离级别分别解决了什么问题？MVCC 是怎么实现的？
- 如果热点行更新冲突严重，你会怎么处理？

### 3.2 Redis
- Redis 为什么快？单线程模型的边界在哪里？
- 缓存穿透、缓存击穿、缓存雪崩分别怎么治理？
- 如何设计一个热点内容缓存方案，既保证命中率又避免脏数据长期滞留？
- Redis 分布式锁为什么容易踩坑？常见问题有哪些？

### 3.3 消息与日志链路
- Kafka 为什么适合做异步削峰？它和 RocketMQ 在关键设计上有什么差异？
- 如何保证消息不丢、不重、可追踪？
- 如果内容审核链路是异步多阶段处理，如何设计消息重试、死信和补偿机制？

## 4. 分布式系统与高并发

### 4.1 分布式基础
- 什么是分布式事务？常见方案有哪些，为什么很多场景更偏向最终一致性？
- 服务发现、配置中心、熔断、限流、降级分别解决什么问题？
- 如果一个核心依赖超时明显升高，你的上游服务该如何自保？

### 4.2 高并发设计
- 如何设计一个支持高并发评论 / 点赞 / 弹幕的后台系统？
- 秒杀、热点活动、直播互动等流量尖峰场景下，哪些环节最容易成为瓶颈？
- 读多写少和写多读多系统，在缓存、分库分表和异步化上的设计差异是什么？
- 如何做接口级限流？令牌桶和漏桶分别适合什么场景？

### 4.3 一致性与可用性取舍
- 为什么很多互联网系统愿意接受短时间不一致，而不是强一致？
- 幂等在后端系统里为什么重要？接口幂等和消息幂等分别怎么做？
- 如果出现重复消费导致业务重复扣减，你会如何修复并预防？

## 5. 系统设计题

### 5.1 内容平台常见系统设计
- 设计一个评论系统，需要考虑哪些表结构、缓存、审核和排序策略？
- 设计一个点赞系统，如何处理热点内容和计数准确性问题？
- 设计一个站内消息 / 推送系统，如何保障实时性和送达率？
- 设计一个用户关注流 / 信息流后台，写扩散和读扩散该怎么权衡？

### 5.2 稳定性与治理
- 如何设计多机房部署与故障切换方案？
- 如果某一机房网络抖动，你会如何避免全站雪崩？
- 监控体系怎么设计才不只是“看 CPU 和内存”？
- 线上容量评估要看哪些核心指标？如何做压测和扩容预案？
