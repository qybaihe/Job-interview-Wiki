# 题库｜腾讯 / 微信事业群 / 基础平台 / 后端开发

## 岗位方向
- 典型基础平台后端场景：通用服务治理、账号/权限、消息基础设施、配置中心、链路追踪、服务稳定性与容量治理
- 常见考察重点：计算机基础、操作系统、网络协议、分布式系统、数据库设计、代码质量、故障处理与平台化抽象能力

## 高频题
1. TCP 三次握手和四次挥手分别解决什么问题？
2. TIME_WAIT 为什么存在？服务端遇到大量 TIME_WAIT 怎么处理？
3. 进程和线程的区别是什么？用户态和内核态如何切换？
4. 死锁发生的必要条件是什么？线上如何识别与处理死锁问题？
5. MySQL 隔离级别分别解决什么问题？MVCC 是怎么工作的？
6. Redis 持久化机制有哪些？AOF 和 RDB 怎么选？
7. 分布式锁常见实现方式有哪些？基于 Redis 的分布式锁有哪些坑？
8. 服务治理中限流、熔断、降级分别是什么？
9. 一致性哈希适合用在哪些场景？
10. 微服务拆分时如何控制链路复杂度？

## 项目题
1. 讲一个你做过的平台化项目，它解决了哪些重复建设问题？
2. 你做过的配置中心/消息中间件/公共服务有没有遇到稳定性事故？怎么处理？
3. 如果一个基础服务 SLA 很高，但业务方调用方式很混乱，你如何治理？
4. 如何推动团队把“救火式开发”转向标准化工程能力建设？
5. 你有没有做过性能压测？指标体系如何设计？

## 系统设计题
1. 设计一个统一配置中心，要求灰度发布、回滚、审计可追踪。
2. 设计一个 RPC 服务治理平台，支持注册发现、限流熔断、监控告警。
3. 设计一个高可用消息投递平台。
4. 设计一个权限系统，支持角色、资源、操作维度授权。
5. 设计一个调用链追踪系统，帮助快速定位线上故障。
