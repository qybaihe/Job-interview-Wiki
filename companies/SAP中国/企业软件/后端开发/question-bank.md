# SAP中国｜企业软件｜后端开发｜question-bank

> 说明：本题库面向 `SAP中国 / 企业软件 / 后端开发`。
> 本轮未检索到足够稳定、可逐题回链的“SAP中国后端真实面经”公开中文单帖，因此采用 SAP 官方公开技术能力域 + 企业软件后端高相关问题的保守式题库构造。
> 所有问题均可映射到 SAP 官方公开文档中的能力主题，但不伪称为单场真实面试原题逐字整理。

## 适用方向
- 企业软件后端开发
- Java / 云应用 / 平台研发
- 企业 SaaS / ERP / 集成平台 / BTP 相关研发

## 一、企业软件建模与业务一致性

1. 企业软件后端和消费互联网后端最大的差异是什么？为什么前者更强调流程一致性、可审计性和跨部门协同？
2. 如果一个业务流程跨采购、库存、财务三个域，事务边界应该如何划分？哪些地方更适合补偿而不是强事务？
3. 为什么 ERP / CRM / SCM 类系统通常更强调主数据一致性，而不是单点极致吞吐？
4. 什么是主数据（Master Data）？如果客户主数据、物料主数据在多个系统之间不一致，会带来哪些后果？
5. 在订单、发票、付款、审批流这些企业流程中，为什么幂等性往往比“接口成功一次”更重要？
6. 如果一次业务操作会触发多个下游系统更新，你会优先选择同步调用、事件驱动还是混合模式？为什么？

## 二、多租户 SaaS 与标准产品/客户定制

7. 多租户 SaaS 常见隔离模式有哪些？shared schema、separate schema、separate database 各适合什么阶段和客户类型？
8. 多租户系统中，租户隔离除了数据库隔离，还需要在哪些层面保证隔离：缓存、对象存储、任务、日志、消息？
9. 为什么企业客户做二开（customization）时，平台架构很容易被“单个大客户需求”带偏？怎么避免把标准产品做烂？
10. 如果客户要求大量字段级扩展和流程差异化，你会如何设计扩展点，而不是直接污染主模型？
11. 如何平衡“标准化产品能力”和“大客户定制诉求”？哪些能力适合配置化，哪些应该坚持代码级实现？
12. 当一个 SaaS 平台要支持租户级配置、租户级版本差异和租户级功能开关时，如何避免出现难以测试的组合爆炸？

## 三、授权、审计与合规

13. 企业软件为什么比很多互联网业务更重视权限模型、审计日志和变更追踪？
14. 设计一套 RBAC + 数据权限系统时，需要哪些核心实体、授权关系和校验时机？
15. 服务级授权和数据级授权的边界是什么？为什么只做接口级鉴权通常不够？
16. 如果同一用户既能通过 UI 操作，又能通过 API/集成接口修改业务数据，审计系统应如何统一记录？
17. 设计一个审计日志系统时，怎样做到“谁在什么时候改了什么、改前改后是什么、通过什么入口改的”都可追踪？
18. 当客户要求删除或修改历史业务数据时，为什么很多企业系统不能简单物理删除？软删除、归档、版本化各适合什么场景？

## 四、企业集成与事件驱动

19. 企业软件为什么通常需要同时对接 ERP、OA、CRM、消息系统和第三方平台？接口层如何设计才能兼顾扩展性与稳定性？
20. 事件驱动架构在企业软件里适合哪些场景？哪些场景更适合同步事务或显式补偿机制？
21. 使用消息中间件做系统解耦时，如何处理重复消息、乱序消息和下游消费失败？
22. 如果一个集成链路里“业务已成功、消息未送达”或“消息已送达、下游处理失败”，你会怎么保证最终可恢复？
23. 为什么对账系统不能只靠“最终一致性”四个字？你还需要哪些补充机制：补偿、对账、重试、人工兜底、审计？
24. 当外部系统接口版本升级导致已有客户脚本或实施方案失效时，如何做兼容治理与发布止损？

## 五、性能、报表与平台治理

25. Java 后端在企业软件里常见的性能瓶颈有哪些？数据库、序列化、规则计算、批处理、权限过滤分别可能出什么问题？
26. 为什么企业软件中的报表查询经常要和主交易库隔离？读写分离、离线数仓、宽表、缓存分别适合什么场景？
27. 企业软件里的批处理和实时处理应该如何分工？哪些链路必须实时，哪些更适合按批次回收？
28. 如果系统要支持多时区、多币种、多语言，数据库建模和应用层设计会受到哪些影响？
29. 灰度发布、蓝绿发布在企业软件里为什么往往比消费互联网更难？
30. 如果你面 SAP中国 的后端岗，在项目介绍里最该重点突出哪些能力：复杂业务建模、企业集成、权限审计、稳定性还是交付落地经验？
