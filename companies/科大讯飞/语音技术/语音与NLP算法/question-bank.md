# 科大讯飞｜语音技术｜语音与NLP算法｜question-bank

> 说明：本题库基于科大讯飞公开技术文档与产品能力说明做岗位能力映射整理。
> 当前更适合作为“语音 / NLP 算法”方向的正式 question-bank，不伪称为单场真实面试逐字题单。

## 岗位方向
- 语音识别（ASR）算法
- 语音合成（TTS）算法
- 语音与文本联合建模 / NLP 算法
- 语音交互、流式处理、工程落地

## 一、语音识别（ASR）基础与工程

1. 科大讯飞语音听写（流式版）为什么采用 WebSocket 长连接，而不是普通短连接 HTTP 轮询？
2. 语音流式识别里，为什么前端通常要按固定帧长切片上传音频？帧长过大或过小分别会带来什么问题？
3. ASR 中端点检测（VAD）的作用是什么？如果 VAD 过早截断，会对识别结果产生什么影响？
4. 流式 ASR 为什么会出现“中间结果”和“最终结果”不一致？你如何向业务解释这种现象？
5. 实时语音识别里，延迟通常拆成哪几段？首字延迟和最终结果延迟分别受哪些因素影响？
6. 如果用户网络抖动导致音频包乱序、丢包、延迟抖动，你会如何在服务端和客户端做容错？
7. 为什么同一个 ASR 模型在安静环境和强噪声环境下表现可能差异很大？你会从哪些方向优化鲁棒性？
8. 语音识别错误通常可分为替换、删除、插入三类，它们分别更可能由哪些问题造成？
9. 中文 ASR 为什么既要关注普通话标准发音，也要关注口音、语速、停顿和数字/英文混读？
10. 如果业务场景是会议转写，和短语音命令词识别相比，模型、解码和后处理策略会有哪些不同？
11. 流式识别中的 partial result 应该如何用于前端展示，才能既“看起来快”又避免频繁闪烁？
12. 如果你来设计 ASR 在线评测体系，会看哪些核心指标：WER/CER、RTF、首包延迟、稳定时间、句终时延？

## 二、语音合成（TTS）与语音交互

13. 在线语音合成为什么也常采用流式返回？它对交互体验的核心价值是什么？
14. TTS 中声学模型和声码器通常分别解决什么问题？为什么这两层经常分开讨论？
15. 如果用户觉得“声音很自然，但情感不对”，你会优先怀疑文本规范化、韵律预测还是说话人建模？
16. 中文 TTS 为什么要特别重视文本规范化（TN）？数字、日期、金额、多音字分别会带来哪些挑战？
17. 语音合成里的停顿、重音、语速、语调控制，分别如何影响用户感知？
18. 如果业务要求支持不同音色、情感和场景模板，TTS 系统设计上应如何抽象 speaker/style 参数？
19. 实时 TTS 里，首包快但后续音频卡顿，可能是哪些链路瓶颈造成的？
20. 在智能客服/数字人场景中，ASR + NLP + TTS 串联后，整条语音交互链路的关键优化点有哪些？

## 三、NLP 与语音结合方向

21. 如果岗位叫“语音 / NLP 算法”，为什么往往不仅考纯 ASR/TTS，还会问文本理解、对话管理和大模型协同？
22. 语音输入转文字后进入 NLP/LLM 模块，ASR 错误会如何向下游传播？你如何降低误识别对理解效果的伤害？
23. 在语音助手场景里，命令识别、意图分类、槽位抽取和开放问答，分别适合什么样的技术路线？
24. 如果语音识别结果缺少标点、断句不稳，下游做摘要、问答、检索会受到哪些影响？如何补救？
25. 你如何理解“语音前端 + 识别 + 文本理解 + 语音合成”是一个整体系统，而不是四个孤立模块？
26. 在中文语音交互里，口语化、省略、指代和纠错重说会给 NLP 模块带来哪些额外难点？
27. 如果要做会议纪要、课堂转写、客服质检这类场景，ASR 和 NLP 的联合优化重点有什么差别？
28. 大模型加入语音交互系统后，哪些问题被改善了，哪些老问题其实依旧存在？

## 四、算法原理高频追问

29. CTC 的核心假设是什么？它为什么适合处理输入输出未对齐的序列建模问题？
30. Attention-based Encoder-Decoder 和 CTC 在 ASR 中各有什么优劣？
31. Transducer（RNN-T）相对 CTC / AED 的优势是什么？为什么它在流式识别里常被提及？
32. Conformer 为什么在语音识别中流行？它相比纯 Transformer 的改进点是什么？
33. SpecAugment 的基本思想是什么？它主要缓解了什么问题？
34. 语言模型在 ASR 中通常如何参与解码？浅融合、深融合的思路有什么区别？
35. 如果领域词很多，比如医疗/教育/车载场景，热词注入或上下文偏置通常怎么做？
36. TTS 中 FastSpeech / Tacotron 类方案分别代表什么思路？各自在哪些约束下更合适？

## 五、系统设计与落地

37. 设计一个流式语音识别服务，支持长连接、音频分片上传、中间结果返回、句终输出和错误恢复。
38. 设计一个语音助手系统，从唤醒、ASR、意图识别、知识问答到 TTS 播报，重点说明延迟预算如何分配。
39. 设计一个支持多场景（会议、客服、车载）的 ASR 平台，如何做模型路由、热词管理和租户隔离？
40. 设计一个 TTS 平台，支持多音色、多情感、多格式输出和调用鉴权，如何保证低延迟与可观测性？
41. 如果识别质量在线上突然下降，你会如何排查：音频输入质量、采样率、编码格式、噪声环境、模型版本、后处理策略？
42. 如果业务要求“端上先出粗结果，云上再精修”，你会如何设计端云协同的语音链路？

## 六、手撕 / 实践追问

43. 手写 Levenshtein Distance，并说明它为什么能用于计算 CER / WER。
44. 给定一串 ASR partial results，如何做稳定拼接，避免把已确认文本反复覆盖？
45. 实现一个滑动窗口/双端队列版本的实时音量检测或简单 VAD。
46. 如果给你一批带时间戳的识别结果，如何做断句和标点恢复的后处理？
