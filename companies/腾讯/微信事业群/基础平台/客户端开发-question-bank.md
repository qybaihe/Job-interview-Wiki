# 题库｜腾讯 / 微信事业群 / 基础平台 / 客户端开发

## 岗位方向
- 典型客户端基础平台场景：通用组件库、网络与容器框架、性能优化、包体治理、跨端能力、稳定性监控、发布与灰度体系
- 常见考察重点：iOS / Android 基础、渲染机制、内存与卡顿治理、网络链路、组件化、跨端方案、工程效率、故障定位能力

## 高频题
1. Android 的 `Activity` / `Fragment` 生命周期各自有哪些关键回调？常见内存泄漏点在哪里？
2. `Handler`、`Looper`、`MessageQueue` 三者之间是什么关系？
3. iOS 的 RunLoop 是什么？它解决了哪些问题？
4. 为什么会出现 UI 卡顿？从主线程任务、绘制、锁竞争、GC / ARC、I/O 等角度怎么排查？
5. 冷启动和热启动分别是什么？启动耗时通常如何拆解与优化？
6. 网络请求从客户端发出到服务端返回，大致经过哪些关键环节？哪些环节最容易影响用户体感？
7. 图片加载框架为什么要做多级缓存？内存缓存、磁盘缓存、解码复用分别解决什么问题？
8. 组件化 / 模块化的收益是什么？什么时候会引入额外复杂度？
9. 崩溃监控、卡顿监控、ANR 监控分别怎么做？
10. 灰度发布、AB 实验、远程配置在客户端基础平台里分别扮演什么角色？

## 算法题

6. 链表反转

## 项目题
1. 讲一个你做过的启动优化项目，问题是怎么发现的，优化后指标如何变化？
2. 如果微信这类高 DAU App 某次版本发布后崩溃率上升，你如何快速止损和定位？
3. 你做过哪些卡顿治理？如何证明不是“玄学优化”？
4. 如果一个公共基础库被很多业务依赖，你如何推动兼容性改造和升级？
5. 讲一个你参与过的网络优化、图片优化或包体治理项目。
6. 如果客户端和服务端接口频繁变更，你会如何设计更稳定的协议与兼容方案？

## 系统设计题
1. 设计一套客户端崩溃监控与上报系统，要求支持版本、设备、场景维度分析。
2. 设计一套卡顿监控方案，要求能区分主线程阻塞、锁竞争和 I/O 问题。
3. 设计一个远程配置平台在客户端的接入方案，要求灰度、回滚、审计可追踪。
4. 设计一个图片加载基础库，要求兼顾性能、内存占用和弱网体验。
5. 设计一个组件化架构，支持业务独立迭代、公共能力复用和问题快速定位。

## 反问方向
- 这个客户端基础平台岗位更偏底层基础设施，还是偏业务通用组件建设？
- 团队当前最重视的指标是启动、卡顿、崩溃、包体，还是工程效率？
- 新人前几个月更可能负责性能治理、基础库维护，还是业务侧接入改造？
- 团队如何平衡跨端方案推进与原生体验保障？
- 对优秀客户端工程师的定义，是性能治理、架构抽象还是跨团队协作推动能力？
