# 联想｜客户端｜系统软件｜question-bank

> 说明：本题库基于联想设备生态、客户端工具、系统级软件常见岗位画像与工作区候选内容整理，定位为 **联想 / 客户端 / 系统软件** 的正式第二方向题库。
> 当前版本不伪称为逐字真实面经，更适合做系统软件 / 终端工具 / 设备管理方向的首版正式入口。

## 一、操作系统与进程线程

1. 进程、线程、协程的差异是什么？系统软件里通常为什么更常直接面对线程模型？
2. Windows 下多线程程序死锁怎么排查？如果现场无法复现，你会保留哪些信息？
3. 用户态和内核态切换为什么有成本？在系统工具软件里哪些场景会放大这种成本？
4. 什么是句柄泄漏？和内存泄漏相比，Windows 客户端里它有什么不同表现？
5. 为什么系统软件常强调状态机设计？相比简单 if-else 流程有什么优势？
6. 后台常驻程序如何降低 CPU、内存和磁盘占用？

## 二、C/C++ 基础与资源管理

7. C++ 智能指针各自适合什么场景？shared_ptr 滥用会带来哪些问题？
8. RAII 的核心价值是什么？为什么系统软件尤其依赖它来管理资源？
9. 如果一个模块频繁崩溃，你会如何区分是越界访问、悬空指针还是并发读写问题？
10. crash dump 常见分析思路是什么？最先看哪些信息最有价值？
11. DLL 动态加载失败常见有哪些原因？
12. 如何做跨版本 ABI 兼容，避免升级后老模块无法正常工作？

## 三、驱动、设备与客户端协同

13. 驱动与用户态程序通信通常有哪些方式？各自适合什么场景？
14. 设备管理软件如何识别不同硬件型号，并适配不同能力开关？
15. 如果设备插拔频繁，客户端如何保证状态同步和 UI 展示一致？
16. 蓝牙、USB、网络设备等多类外设并存时，系统软件层最容易出现哪些资源竞争问题？
17. 为什么很多设备管理工具要维护一套本地缓存状态，而不能每次都实时探测硬件？
18. 如果设备固件升级失败，客户端应如何做回滚和恢复提示？

## 四、安装、升级与可回滚

19. 安装器/升级器如何保证幂等？为什么“重复安装不会把系统搞坏”很重要？
20. 自动更新模块通常包含哪些核心能力：版本检测、灰度、断点续传、校验、回滚？
