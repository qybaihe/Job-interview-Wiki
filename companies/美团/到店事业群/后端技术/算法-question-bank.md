# 题库｜美团 / 到店事业群 / 后端技术 / 算法

## 岗位方向
- 典型算法场景：本地生活到店搜索排序、推荐分发、广告投放、供需匹配、转化预估、优惠券与营销策略优化
- 常见考察重点：机器学习基础、推荐与搜索排序、多目标优化、特征工程、样本偏差、线上实验、业务指标理解、算法工程落地能力

## 高频题
1. CTR / CVR / 核销率预估任务中，样本是如何构造的？曝光、点击、下单、核销标签分别有哪些常见偏差？
2. 推荐系统中的召回、粗排、精排、重排分别解决什么问题？在到店场景里哪一层最容易受供给分布影响？
3. 为什么同一个模型离线 AUC 提升了，线上 GMV、下单率或核销率不一定提升？
4. 样本不均衡时常见处理方式有哪些？过采样、欠采样、加权损失分别适合什么场景？
5. Feature crossing 为什么重要？Wide、FM、DeepFM 在特征交叉上的差异是什么？
6. 什么是数据漂移和概念漂移？线上如何监控？
7. 搜索和推荐在本地生活场景中最大的不同是什么？它们在意图强弱、地理约束和指标设计上有哪些差异？
8. 排序模型延迟预算很紧时，如何在效果和性能之间做取舍？
9. 如何设计线上 AB 实验？实验指标、样本分桶、显著性判断和副作用指标分别怎么做？
10. 如果一个城市的商户供给很稀疏，推荐系统怎么避免“越推越集中”？
11. 优惠券或补贴策略为什么不能只看领取率？转化率、核销率和补贴成本应如何一起看？

## 项目题
1. 讲一个你做过的推荐、搜索或广告排序项目，业务目标和核心收益分别是什么？
2. 如果到店推荐点击率不错，但核销率一直低，你会怎么分析问题？
3. 你如何判断模型优化带来的收益是真的，而不是流量波动或促销活动造成的？
4. 商户冷启动、城市冷启动、活动冷启动分别怎么处理？
5. 如果商户侧强诉求和用户体验冲突，你怎么在排序策略里平衡？
6. 讲一次你做特征治理、样本治理或线上问题排查的经历。
7. 讲一个优惠券或补贴策略优化项目，你如何平衡转化率、核销率和补贴成本？

## 系统设计题
1. 设计一个本地生活到店搜索排序系统，说明召回、排序、特征服务和实验平台如何协同。
2. 设计一个优惠券推荐系统，要求兼顾领取率、核销率和补贴成本。
3. 设计一个商户冷启动策略系统，要求新店能获得合理曝光且不过度伤害整体体验。
4. 设计一个线上特征平台，支持离线训练与在线推理特征一致。
5. 设计一个算法实验平台，支持分桶、指标看板、回滚和异常告警。
