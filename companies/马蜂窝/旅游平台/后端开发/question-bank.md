# 马蜂窝｜旅游平台｜后端开发｜question-bank 候选

> 说明：仅使用公开可访问来源中的可见文本、标题、摘要与公司/岗位讨论片段整理；本候选定位为 **question-bank 补强**，不冒充真实逐题还原面经。
> 
> 证据主轴：
> 1. 马蜂窝属于旅游/社区平台；
> 2. 公开讨论中可见其部分技术栈线索：`云平台`、`docker`、`k8s`、`Go`，同时出现 `Java`、`Python`；
> 3. 因缺少稳定、完整、可核验的单篇后端面经原文，本轮更适合产出旅游平台后端 question-bank，而不是 interview-experiences JSON。

## A. 旅游平台业务理解 / 项目深挖

1. 先做自我介绍，重点讲一个你做过的旅游、电商、内容平台或高并发后端项目。
   - 来源映射：公开求职讨论页普遍围绕岗位选择与技术方向展开；适合作为后端一面开场高频题。
2. 如果让你设计一个“旅游平台”的核心后端，你会如何拆分内容社区、搜索、交易、库存、营销这几块服务？
   - 来源映射：马蜂窝被公开讨论为旅游/社区平台；题目围绕平台型业务合理展开。
3. 旅游平台里“攻略/游记/问答/目的地内容”与“机酒/门票/当地玩乐交易”共存时，数据域边界怎么划分？
   - 来源映射：旅游平台业务特征 + 平台后端岗位常见系统拆分题。
4. 如果你的项目 QPS 翻 10 倍，哪些链路最可能先出问题？你会如何分层止损？
   - 来源映射：平台后端岗位高频项目深挖题。
5. 你做过最复杂的一次线上故障排查是什么？最终是代码、配置、依赖、数据库还是资源瓶颈？
   - 来源映射：云平台 / 后端工程方向常见深挖。

## B. 云平台 / 容器 / Go 工程能力

6. 公开讨论里提到马蜂窝部分岗位“云平台那块，docker、k8s 相关，主要语言是 Go”，如果让你负责这一方向，你会如何描述一个服务在 K8s 上的发布链路？
   - 来源映射：牛客公开搜索结果片段，“马蜂窝是云平台那块，docker,k8s相关。主要语言是go”。
7. Docker 镜像为什么要分层？镜像体积过大时，你通常怎么优化？
   - 来源映射：同上，docker 相关岗位线索。
8. K8s 中 Deployment、StatefulSet、DaemonSet 分别适合哪些场景？
   - 来源映射：同上，k8s 相关岗位线索。
9. 线上服务在 K8s 里频繁 OOMKilled，你会如何从 JVM/Go 进程、容器限额、流量模型三个角度排查？
   - 来源映射：云平台 + 后端排障常见题。
10. Go 的 goroutine 调度模型、channel 使用边界、context 取消传播机制分别是什么？
   - 来源映射：公开讨论片段明确提到“主要语言是 go”。
11. 如果历史系统以 Java/Python 为主，而新平台服务偏 Go，你会如何处理多语言服务的协议、日志、监控、错误码统一？
   - 来源映射：公开讨论片段同时出现 Java、Python、Go。

## C. 高并发后端 / 稳定性治理

12. 如果旅游平台在节假日、热门演唱会城市、突发热点目的地时流量暴涨，你会如何做限流、降级、熔断？
   - 来源映射：旅游平台业务天然峰值明显；适合后端场景题。
13. 令牌桶、漏桶、固定窗口、滑动窗口各自适合什么场景？
   - 来源映射：后端平台岗位高频题；与高峰流量治理强相关。
14. 如何设计一个幂等的下单 / 收藏 / 点赞 / 发帖接口？
   - 来源映射：旅游社区与交易平台通用后端能力题。
15. 缓存击穿、穿透、雪崩分别是什么？在热门目的地页/榜单页里你怎么防？
   - 来源映射：旅游搜索与内容热点场景。
16. 如果某个下游服务 RT 突然拉高，你会优先看哪些指标？
   - 来源映射：后端稳定性和观测性常见题。
17. 如何做灰度发布？如果灰度后发现目的地详情页错误率升高，回滚策略怎么定？
   - 来源映射：云平台 / K8s / 发布治理常见题。

## D. 数据库 / 存储 / 一致性

18. MySQL 索引为什么能加速查询？联合索引最左匹配在真实业务里怎么体现？
   - 来源映射：后端基础高频题。
19. 行锁、表锁、间隙锁、临键锁分别解决什么问题？
   - 来源映射：后端面试高频数据库题。
20. 旅游平台里库存、订单、支付状态更新时，如何避免超卖与脏写？
   - 来源映射：旅游交易场景的强相关问题。
21. 分库分表后，跨分片分页、排序、去重怎么做？
   - 来源映射：平台型后端高频扩展题。
22. Redis 适合存什么，不适合存什么？如果把热门攻略榜单放 Redis，你怎么设计过期与重建？
   - 来源映射：内容/旅游平台热点读场景。
23. MQ 在旅游平台里最适合放在哪些链路？为什么不全都用同步调用？
   - 来源映射：交易、通知、异步削峰场景通用。

## E. 搜索 / 推荐 / 内容平台场景

24. 马蜂窝公开讨论里还能看到“搜索”“推荐算法”等岗位线索；如果你负责旅游搜索后端，query 改写、召回、排序、缓存应该怎么协同？
   - 来源映射：牛客公开搜索结果出现“马蜂窝搜索”“推荐算法岗”等文本片段。
25. 一个目的地搜索服务，如何处理热门词、错别字、同义词、拼音、英文地名？
   - 来源映射：旅游搜索场景强相关。
26. 如果首页推荐既要做内容分发，又要兼顾交易转化，你会如何平衡 CTR、CVR、GMV 等目标？
   - 来源映射：公开讨论存在“推荐算法岗”线索；适合作为平台跨团队协作题。
27. 旅游攻略/游记存在大量图文内容，如何做内容审核、去重和违规处理链路？
   - 来源映射：旅游社区平台属性。
28. 详情页里同时有图文内容、评论、相关推荐、实时价格时，聚合接口应该怎么做，才能兼顾性能和可维护性？
   - 来源映射：内容+交易混合详情页的典型后端题。
