## 一、岗位画像与面试特征

从京东物流官网与招聘站可归纳出，该方向更像“供应链基础设施 + 物流平台后端”：

- 业务覆盖快递、快运、仓配一体、送仓、冷链、医药、国际等多条链路；
- 强调“技术驱动”的供应链解决方案及物流服务商定位；
- 物流科技板块明确提到：
  - 自动化、数字化、智能化
  - 多智能体协同
  - 智能仓储
  - 智能物流园区
  - 数智化供应链
- 仓配/送仓/冷链等页面提供了大量可落到后端设计的问题线索，例如：
  - 大促千万级单量稳定运行
  - 分仓调度
  - 多仓/多货主/多平台
  - 全程可视化轨迹
  - 多温层控温
  - 正逆一体

因此，京东物流后端面试更可能是：

- 先问常规后端基础（Java / 并发 / MySQL / Redis / MQ / Spring）
- 再追问你如何把这些能力落到物流平台场景中
- 重点考查系统设计、稳定性、容量治理、链路协同和业务抽象能力

## 二、可直接入库的候选问题（30 题）

### A. 平台基础与高并发题

1. 如果一个云仓系统需要支撑**大促千万级单量稳定运行**，你会如何设计核心链路的限流、削峰、异步化和容量预案？
2. 面对**多行业、多平台、多货主、多仓库**的仓储 SaaS 场景，后端的数据模型如何设计，才能兼顾隔离性与可扩展性？
3. 对于云物流订单管理系统，订单、库存、履约、逆向单据之间如何做领域拆分，避免一个“大订单表”承载所有语义？
4. 如果仓储作业管理系统需要支持多种作业流：入库、上架、拣货、复核、打包、出库，你会如何设计状态机与作业编排？
5. 在多租户 SaaS 系统中，如何做租户级别的限流、数据隔离、权限隔离和任务优先级控制？
6. 如果物流查询链路要求“全程可视”，你会如何设计轨迹写入、轨迹查询与轨迹回放能力？

### B. 仓配一体 / 分仓调度场景题

7. 京东物流公开资料提到**分仓调度**，如果你负责分仓系统，核心输入、约束条件和目标函数会如何抽象？
8. “智能分货补货”场景下，如何设计库存分布、需求预测、履约时效和跨区成本之间的权衡逻辑？
9. 对于“就近一站交付全量库存、目的仓间自动调拨”的智能商务仓场景，如何保证调拨计划与实际库存的一致性？
10. 仓配一体场景里，订单履约、库存占用、出库波次和配送计划的事件顺序可能乱序到达，如何保证最终状态正确？
11. 如果一个仓配系统要支撑**10 倍+ 高弹性订单波峰**，数据库、缓存、MQ、任务系统分别应该如何扩容和降级？
12. 如何设计“正向仓配 + 逆向售后”统一平台，让逆向退货、质检复原、快速出库不会把正向履约链路拖垮？

### C. 送仓 / 干支配 / 运力协同题

13. 京东送仓服务公开信息提到**90多个转运中心、200多个提货城市、500多个交仓团队、5000多辆运输车辆**。如果你设计送仓平台，线路、运力、波次、交仓预约应如何建模？
14. 面对“货源地 → 暂存 → 分仓调度 → 运输 → 送仓 → 退仓”的一体化链路，如何设计事件总线与跨系统状态同步机制？
15. 送仓场景中存在小批量快速补货、大票直送、整车专送等不同模式，后端如何抽象统一订单模型，同时保留差异化规则？
16. 如果要求“送仓上架准时、异常 1 小时响应、全程跟踪”，你会设计哪些 SLA 指标、告警机制与异常补偿流程？
17. 对于“专车直发无中转”和“零担多站点流转”两种模式，轨迹系统和时效预测系统应如何分别建模？

### D. 冷链 / 医药 / 可追溯场景题

18. 冷链业务存在**多温层控温、温度轨迹可视、化冻赔付**要求。后端如何设计温控数据采集、异常告警与履约责任归因？
19. 在冷链卡班/城配/整车多模式并存时，如何统一运输任务模型，同时支持不同温层、不同时效、不同计费规则？
20. 医药物流要求全程追溯、温湿度监控、数据长期存储。若你负责后端平台，如何设计审计日志、监管留痕和数据不可篡改能力？
21. 对于生鲜/医药这类高价值时效业务，若轨迹更新延迟或温度数据缺失，你会怎样设计补采、补传与降级展示策略？

### E. 智能仓储 / 物流科技平台题

22. 京东物流公开资料提到智能仓储依赖**自主研发的软件系统、核心智能设备、算法优化、数据分析、系统仿真能力**。如果你做平台后端，如何划分“设备接入层、业务层、算法层、仿真层”的边界？
23. 当仓内机器人、输送线、WMS、调度系统同时上报事件时，如何设计统一事件模型，避免设备协议差异污染业务域？
24. 多智能体协同场景下，如果算法服务给出路径/任务分配建议，但现场设备状态实时变化，后端如何支持动态重规划？
25. 智能物流园区强调“一体化园区规划和数字化能力”，如果要做园区级实时看板，你会如何组织仓、车、货、场、人的核心指标？
26. 如果要建设一个服务多个业务线的物流科技平台，哪些能力适合沉淀为平台层：账号、租户、订单中心、轨迹中心、规则引擎、调度引擎、报表中心？为什么？

### F. 项目深挖与系统设计追问

27. 你做过的项目里，如果要迁移到京东物流这种多仓、多链路、多履约模式场景，原方案最先会在哪些点失效？
28. 设计一个“物流状态消息总线”，接入快递、仓储、运力、冷链、售后等多个上游系统，并向查询、履约、客服、商家侧多个下游分发事件。
29. 设计一个“物流轨迹查询系统”，既要支持用户高频查询，也要支持内部运营排障、异常回放和责任归因。
30. 设计一个“云仓 SaaS 平台”，要求支持多租户、多仓、多货主、多平台接入，并能在大促期间稳定支撑千万级单量。
