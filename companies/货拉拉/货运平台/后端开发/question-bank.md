# 货拉拉｜货运平台｜后端开发｜question-bank

> 说明：本题库基于货拉拉官网公开业务信息与公开面经线索整理，优先保守落为正式 `question-bank`。
> 证据边界：
> - 可稳定确认公司业务形态：同城/跨城货运、企业物流、搬家、零担、跑腿、冷运、车后市场、多车型智能调度；
> - 可作为“货运平台 / 后端开发”方向的公开业务映射题库；
> - 不伪称为单篇真实逐轮面经，也不直接包装成 `interview-experiences/*.json`。

## 岗位方向
- 后端开发
- 服务端研发
- 货运平台 / 交易履约
- 调度、计价、订单、运力、企业物流

## 公开业务背景映射
- 货拉拉官网公开显示其为“互联网+”综合性物流平台；
- 业务覆盖同城/跨城货运、企业版物流、搬家、零担、跑腿、冷运、汽车租售及车后市场；
- 平台依托移动互联、大数据和人工智能技术，支持多种车型即时智能调度；
- 这意味着其后端高频问题通常会集中在：订单状态机、运力调度、计价规则、异步履约、企业物流、多城市多车型资源编排、风控与稳定性。

## 一、平台通用后端能力
1. MySQL 索引下推、覆盖索引、回表分别是什么？在订单表 / 运单表 / 计价明细表这种宽表里如何设计索引？
2. Redis 在货运交易链路里适合放什么数据，不适合放什么数据？缓存与数据库一致性怎么做？
3. MQ 在货运平台里一般用在哪些地方？消息重复、乱序、积压怎么处理？
4. 分布式事务在订单、司机抢单、计价、优惠券、支付之间如何取舍？你会选最终一致性还是强一致？
5. 如何设计一个幂等接口？下单、取消、支付回调、派单回执分别怎么做幂等？
6. 线程池参数怎么定？如果晚高峰线程池被打满会发生什么？
7. JVM Full GC 频繁时，在线交易服务如何排查并止损？
8. Spring 事务在自调用、异常吞掉、异步线程里为什么容易失效？
9. 分库分表后，订单按用户维度、司机维度、运单维度查询如何兼顾？
10. 如何设计高可用的状态机系统？

## 二、货运平台 / 订单履约专题
11. 货拉拉这类平台的订单状态机通常会比普通电商多哪些状态？例如待接单、已接单、司机到达、装货中、运输中、卸货中、已完成、异常取消。
12. 如果用户下单后长时间无人接单，系统应该如何做重试派单、加价建议和失败兜底？
13. 司机接单后又取消，订单状态、调度任务和补偿策略应该怎么设计？
14. 订单取消在“未接单 / 已接单待到达 / 装货中 / 运输中”四个阶段分别如何处理？
15. 如何设计一个货运平台交易中台，让订单、计价、支付、履约、售后都围绕统一运单号和状态机工作？
16. 如果同一城市短时间涌入大量下单请求，交易服务如何限流、削峰和降级？
17. 如何处理“用户看到可接单运力充足，但实际上附近司机资源已被占用”的问题？
18. 货运平台里的“下单成功”与“履约成功”为什么是两个不同层次的问题？

## 三、调度 / 运力 / 路径规划专题
19. 货拉拉官网强调多车型即时智能调度。后端如何抽象车型、容积、载重、距离、时段、限制路段等多维约束？
20. 如果同一单既可以由面包车承接，也可以由货车承接，调度系统如何做车型推荐与成本平衡？
21. 调度系统为什么通常要区分预估派单、实时派单和重试派单三类任务？
22. 如果司机端定位有延迟或漂移，调度系统会受到什么影响？如何做纠偏？
23. 如何设计 ETA（预计接驾 / 预计送达）服务？核心输入特征有哪些？
24. 当天气、拥堵、限行等外部因素突变时，路线规划和调度策略如何做到分钟级更新？
25. 如果一个司机同时被多个订单竞争，系统如何做锁定、超时释放和二次分配？
26. 实时调度场景里为什么不能把所有决策都放在数据库事务里完成？

## 四、计价 / 支付 / 结算专题
27. 货运平台的计价为什么通常比普通网约车更复杂？哪些因素会进入报价：车型、距离、楼层、装卸、等待、夜间、跨城、高速费等？
28. 如何设计一个可配置的计价引擎，让业务可以按城市、车型、时间段灵活调价？
29. 如果报价展示给用户后，履约中又出现等待费、搬运费、附加费，系统如何保证价格解释性与审计能力？
30. 支付回调、补差价、退款、优惠券核销同时发生时，如何保证账务一致性？
31. 企业物流客户和 C 端个人用户在结算链路上最大的系统差异是什么？
32. 如果企业客户需要月结、账期、对账单、发票能力，账务系统应该如何扩展？

## 五、企业物流 / B端平台专题
33. 官网公开有企业版物流。企业物流后端为什么通常更强调组织、权限、审批流和对账，而不只是即时下单？
34. 如果一个企业客户有多个门店、多个下单人、多个成本中心，系统如何建模组织和权限？
35. 企业物流 API 开放给商家 ERP/WMS 后，开放平台鉴权、限流、审计应该怎么做？
36. B 端客户要求 SLA 报表、运输时效、司机响应率、异常单分析时，指标平台应该怎么设计？
37. 为什么企业物流产品通常需要把“标准产品能力”和“大客户定制诉求”分开治理？

## 六、稳定性 / 风控 / 系统设计题
38. 如果黑产利用脚本批量下虚假搬家单或恶意占用司机资源，风控系统应该关注哪些信号？
39. 调度平台、订单平台、支付平台之间出现消息重复消费时，如何保证不重复派单、不重复扣费、不重复结算？
40. 节假日/雨天/搬家旺季来临前，货运平台最该压测哪些链路？看哪些指标？
41. 如果某城市的调度服务故障，但用户下单不能完全不可用，你会怎样设计降级方案？
42. 设计一个支持多城市、多车型、多约束条件的货运调度平台。
43. 设计一个货运平台订单状态机系统，支持异常取消、改价、司机改派和售后。
44. 设计一套企业物流对账与月结系统，支持账期、开票、异常核销和对账追踪。
45. 设计一套运力供给保护系统，防止热点区域被瞬时抢空并支持司机公平分配。
46. 设计一个货运平台的可观测体系，覆盖派单成功率、接单时延、履约完成率、异常单率和计价误差。
