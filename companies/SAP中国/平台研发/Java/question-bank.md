# SAP中国｜平台研发｜Java｜question-bank

> 说明：本题库基于公开可审计的 SAP 技术能力域与企业软件平台研发语境整理，定位为 **SAP中国 / 平台研发 / Java** 的正式 question-bank。
> 当前版本不伪称为单场真实面经逐字题单，适合用作 SAP 中国第二方向的正式补充入口。

## 适用方向
- 平台研发 / Java
- 企业软件平台后端
- SaaS 平台、多租户、流程与集成研发

## 一、Java 基础与并发

1. HashMap 在 JDK7 和 JDK8 中的核心实现差异是什么？为什么扩容和并发下的问题常被追问？
2. synchronized、ReentrantLock、CAS 分别适合什么场景？为什么 CAS 不能解决所有并发问题？
3. 线程池核心参数有哪些？如果平台服务线程池被打满，你会如何定位问题根因？
4. ThreadLocal 为什么“方便但危险”？在线程池复用场景下有哪些坑？
5. Java 中常见的内存泄漏场景有哪些？企业平台服务如何做排查？
6. JVM 调优通常先看哪些指标？GC 问题和慢 SQL、锁竞争如何区分？

## 二、Spring / 事务 / 数据一致性

7. Spring Bean 生命周期大致分几个阶段？AOP 和事务代理通常在什么位置介入？
8. 事务传播机制有哪些？企业软件里哪些场景更适合拆成补偿而不是强事务？
9. MySQL 索引失效的常见场景有哪些？为什么很多“加了索引还是慢”的问题本质不在索引本身？
10. 如果一个流程跨审批、主数据、审计三个服务，如何做最终一致性设计？
11. 为什么企业软件里接口幂等经常比“调用成功一次”更重要？
12. 如果下游系统处理成功但消息发送失败，你会如何做补偿与对账？

## 三、多租户 SaaS 与平台抽象

13. 多租户系统常见的隔离模式有哪些？shared schema、separate schema、separate database 各适合什么阶段？
14. 为什么企业平台很容易被“大客户定制需求”带偏？如何通过扩展点设计守住平台边界？
15. 如果要支持租户级配置、租户级功能开关和租户级版本差异，如何避免组合爆炸？
16. 平台研发和普通业务研发最大的差异是什么？为什么平台更强调抽象、复用和治理？
17. 如何设计插件化扩展机制，让不同客户能扩展流程但不污染主干模型？
18. 为什么企业平台很重视 schema 演进和兼容治理？

## 四、权限、审计与工作流

19. RBAC 和 ABAC 的差异是什么？在企业软件里各自适合什么层级？
20. 仅做接口级鉴权为什么通常不够？数据级权限一般在哪里校验更合理？
