# 线程执行规则（必须严格执行）

## 规则 1：每个线程一次只做 1 家公司
不允许：
- 一个线程同轮处理 3~5 家公司
- 一个线程既补老公司又做新公司
- 一个线程同时写 question-bank 和多个 JSON 方向

必须：
- 1 个线程 = 1 家公司
- 该线程本轮目标就是为这家公司拿到首个正式结果

## 规则 2：优先拿“首个正式入口”
对新公司来说，最优先的是：
- `companies/<公司名>/<方向>/question-bank.md`

只要质量够、来源够稳，允许先用 question-bank 破冰。
不要为了追一份不稳的真实 JSON 把整轮拖死。

## 规则 3：先覆盖公司，再补深度
在 50 家池子没有基本铺开前：
- 老公司补强 = 次优先
- 治理类任务 = 兜底
- 模板清理 / 统计修正 = 最低优先

## 规则 4：每轮尽量有提交
如果本轮多个线程都没法拿强结果，优先收口其中最接近正式落库的一家公司，确保：
- 每轮尽量有正式 commit
- 每轮尽量让公司池继续扩大

# 本阶段的判断标准

## 一轮成功
满足任一即可：
1. 新增 1 家此前未覆盖公司，并正式落 1 个方向文件
2. 新增 2 家以上候选公司，并形成下一轮可直接收口的强候选包
3. 多线程里至少 1 条线程正式落库、其余线程形成规范候选包

## 一轮不算真正成功
以下情况不应自我包装成“高质量成功”：
- 只补老公司题库
- 只改 README / progress_state
- 只有 report，没有正式新增公司
- 一个线程同时研究多家公司但都没落库

在公司数量没有做大之前，不要重新退回“少数公司精修模式”。
