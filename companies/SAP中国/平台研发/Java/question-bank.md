# SAP中国｜平台研发｜Java｜question-bank

> 说明：本题库基于公开可审计的 SAP 技术能力域与企业软件平台研发语境整理，定位为 **SAP中国 / 平台研发 / Java** 的正式 question-bank。
> 当前版本不伪称为单场真实面经逐字题单，适合用作 SAP 中国第二方向的正式补充入口。

## 适用方向
- 平台研发 / Java
- 企业软件平台后端
- SaaS 平台、多租户、流程与集成研发

## 一、Java 基础与服务端工程

1. HashMap 在 JDK7 和 JDK8 的核心差异是什么？为什么扩容、链表转红黑树、并发安全问题在 Java 面试里总被追问？
2. synchronized、ReentrantLock、CAS 分别适合什么场景？CAS 为什么不能解决所有并发问题？
3. 线程池核心参数有哪些？如果平台服务线程池被打满，你会从哪些指标入手排查？
4. ThreadLocal 在线程池场景下有哪些隐患？为什么它在企业平台里“好用但危险”？
5. Java 服务常见的内存泄漏来源有哪些？如何结合堆 dump、GC 日志和对象引用链定位？
6. JVM 调优时你通常先看哪些指标？如何区分是 GC、锁竞争、CPU 打满还是数据库慢查询导致的延迟抖动？

## 二、事务、一致性与数据层

7. Spring 事务传播机制有哪些？企业软件里哪些链路适合强事务，哪些更适合补偿或异步解耦？
8. MySQL 索引失效的常见场景有哪些？为什么很多“加了索引还是慢”的问题本质不在索引本身？
9. 如果一个业务流程跨审批、主数据、审计三个服务，事务边界应该如何划分？
10. 为什么企业软件里的接口幂等经常比“调用成功一次”更重要？
11. 如果下游系统处理成功但消息发送失败，你会如何设计补偿、重试与对账机制？
12. 企业软件中的报表查询为什么常常要与主交易库隔离？读写分离、宽表、离线数仓分别适合什么场景？

## 三、多租户 SaaS 与平台抽象

13. 多租户 SaaS 常见隔离模式有哪些？shared schema、separate schema、separate database 各适合什么阶段？
14. 为什么多租户平台不能只做数据库层隔离？缓存、日志、消息、任务上下文还要怎样做 tenant-aware 设计？
15. provider tenant 和 subscriber tenant 分别承担什么职责？租户订阅与开通链路中最容易出现哪些一致性问题？
16. 如果某个租户升级失败，怎样设计“租户级升级 / 回滚 / 重试”机制，避免影响其他租户？
17. 为什么企业平台容易被“大客户定制需求”带偏？如何通过扩展点机制守住标准产品主干？
18. 如何设计插件化或配置化扩展机制，让不同客户能扩展流程但不污染主干领域模型？

## 四、请求上下文、权限与审计

19. 在 Java 平台服务里，为什么 request context / tenant context / user context 不能只靠普通 ThreadLocal 想当然地传？在线程池复用、异步回调、消息消费时最容易出什么错？
20. 如果一个同步请求里拿到了 tenant 和 user 信息，但后续要异步落库、发消息、写审计日志，你会怎样设计上下文透传，避免出现“数据在 A 租户，日志却记到 B 租户”的事故？
21. RBAC 和 ABAC 的差异是什么？企业软件里各自适合解决什么层级的问题？
22. 为什么只做接口级鉴权通常不够？数据级授权更适合落在查询过滤、领域服务还是网关层？
23. 基于模型声明的授权规则，为什么通常比把鉴权逻辑散落在 controller/service 里更稳？
24. 审计日志为什么不能等同于普通业务日志？它至少需要回答哪些问题：谁、何时、通过什么入口、修改了什么？
25. 为什么企业系统中“删除历史业务数据”往往比新增更敏感？软删除、归档、版本化分别适合什么场景？
26. 如果同一用户既能通过 UI 操作，又能通过 API / 集成接口修改业务数据，审计系统应如何统一记录？

## 五、事件驱动、Outbox、集成与工作流

27. topic 和 queue 分别适合解决什么问题？为什么企业集成里通常两者都要理解？
28. 为什么 transactional outbox 比“先写库再直接发消息”更可靠？它本质上是在避免哪两类典型事故？
29. 如果消息已写入 outbox 但投递失败，你会怎样做补偿、重试和可观测性治理？
30. 企业软件里的事件驱动为什么更强调幂等消费、去重和人工兜底，而不只是吞吐量？
31. 工作流引擎通常包含哪些核心能力？流程定义、节点执行、回退、转派、催办应怎样抽象？
32. 如果让你设计一个既支持标准审批、又支持租户级自定义节点的流程平台，你会如何拆模型边界？
33. 当外部系统接口版本升级导致已有客户脚本或实施方案失效时，如何做兼容治理与发布止损？
34. 如果面 SAP中国 的平台研发 / Java 岗，你会如何用一个项目证明自己既懂 Java 工程实现，又懂多租户、授权和企业集成这三类企业软件复杂度？
